# Stock Decision Pro V1.2

A professional, mobile-first, bilingual Streamlit web app for explainable stock research. Users can search by ticker or company name, choose **Swing Trading** or **Investment**, and see a quick decision first with optional evidence and charts.

> **Important:** This is a research and education tool, not personalized financial advice. It does not guarantee returns. Market data can be delayed, incomplete, or temporarily unavailable.

## What is included

- Arabic and English language selection on first launch.
- One-click language switching after launch.
- Arabic right-to-left layout and mobile-first responsive styling.
- Ticker/company autocomplete using a bundled list plus `yfinance.Search` when available.
- Separate deterministic engines:
  - **Swing Trading:** trend, momentum, liquidity, entry quality, room to resistance, and risk/reward.
  - **Investment:** growth, profitability, financial health, valuation, quality, and technical timing.
- Quick result card with:
  - decision;
  - score and confidence;
  - current price and daily change;
  - entry zone, risk level, and price targets.
- Expandable details for reasons, risks, blockers, score breakdown, indicators, fundamentals, methodology, and charts.
- Interactive daily candlestick chart with volume, EMA 20, SMA 50, SMA 200, support, and resistance.
- Cached requests and graceful data-error handling.
- Transparent source code and unit tests for the calculation layer.
- A bilingual **Evaluation Center** with conservative point-in-time swing backtesting.
- Adjustable historical period, target, holding period, entry window, and slippage.
- Target-hit rate, average return, profit factor, drawdown, model return, stock buy-and-hold, and SPY reference.
- Score-band calibration, sequential equity curve, complete signal ledger, and CSV export.
- A downloadable in-session log of recommendations generated by the live analysis screen.
- A saved **fixed-universe portfolio report** covering the 149 bundled securities.
- Portfolio-aware trade acceptance with cash, 10-position capacity, 1% maximum stop risk, and 10% position caps.
- Universe-level equity curve, score calibration, per-security table, rejected-capacity count, and downloadable accepted-trade ledger.

## Run locally

Python 3.11 or 3.12 is recommended.

```bash
python -m venv .venv
source .venv/bin/activate          # macOS / Linux
# .venv\Scripts\activate           # Windows
pip install -r requirements.txt
streamlit run app.py
```

Open `http://localhost:8501`.

Upload the **contents of this project**, not the ZIP itself. The repository root must contain `app.py` and `requirements.txt`.

## How the decisions work

The app does not use an LLM to invent a recommendation. Each engine applies visible rules and weighted scoring. Missing data lowers confidence, and material blockers can prevent an entry result.

### Swing Trading — 100 points

| Component | Weight |
|---|---:|
| Trend and market context | 25 |
| Momentum | 15 |
| Volume and liquidity | 15 |
| Entry quality | 15 |
| Room to the selected target | 15 |
| Risk/reward | 15 |

Potential blockers include insufficient history, low average dollar volume, inadequate room before estimated resistance, weak risk/reward, and a possible earnings event within five days.

### Investment — 100 points

| Component | Weight |
|---|---:|
| Revenue and earnings growth | 20 |
| Profitability and free cash flow | 20 |
| Financial health | 15 |
| Valuation | 20 |
| Business quality | 15 |
| Technical timing | 10 |

Investment results depend heavily on available company fundamentals. Limited coverage forces a lower-confidence **Wait** result rather than pretending the data is complete.

## Regenerate the fixed-universe test

The report is generated outside Streamlit so visitors do not wait several minutes or overload the free server:

```bash
python scripts/run_universe_backtest.py --period 5y --target 5 --workers 8
```

Default portfolio assumptions are $100,000 initial capital, at most 10 simultaneous positions, no more than 1% equity risk at the published stop, a 10% maximum position weight, three sessions to fill the entry, ten sessions maximum holding, and 0.10% slippage on each side.

The bundled V1.2 report tested all 149 current-list securities and accepted 1,061 of 1,310 executable candidates after capacity constraints. Its result was approximately **-1.00%**, with a **0.99 profit factor**, **21.0% target-hit rate**, and **-12.16% realized maximum drawdown**. This unfavorable result is intentionally retained: it shows that the current recommendation thresholds require improvement and should not yet be relied upon for live trading.

The report uses today's fixed list over historical dates. That creates current-universe/survivorship bias. The next research phase should use historical index membership, delisted securities, point-in-time earnings calendars, and a separate untouched out-of-sample period.

## Data-source limitations

This V1.2 uses `yfinance`/Yahoo Finance for prototyping. It is convenient but is not a licensed commercial market-data feed. Before charging users or relying on real-time execution:

- replace it with a licensed provider;
- add exchange-specific delay labels;
- store recommendation snapshots and immutable input data;
- run the evaluation across a fixed multi-stock universe and separate out-of-sample periods;
- connect a point-in-time historical earnings calendar and historical fundamentals;
- persist live recommendation snapshots in a durable database;
- add monitoring, rate-limit protection, and data-quality checks;
- obtain appropriate legal and regulatory review for the intended markets.
